📱 "Çözüm Sende" Hackathon - Rules and Submission Guide

🧾 General Overview

  • ⏱ Duration: May 24, 2025 – 19:00 ⏳ May 25, 2025 – 11:00 (16 hours)
  • 📍 Format: On-site event. Project management and submissions via GitHub.
  • 💡 UI/UX: All designs must be prepared using Figma. Development via GitHub.
  • 🔧 Technologies: All used must be clearly stated in the README file.
  • 📂 Repository: Private GitHub repo named cubehack_projectname.
  • 📝 Documentation: All docs, setup, and team details in README.
  • 🎥 Demo: Short demo video and Figma prototype are mandatory.

📥 Repository Setup Instructions (IMPORTANT!)

  1. Fork this repository to your personal GitHub account
  2. Rename the forked repository to cubehack_projectname
  3. Set repository visibility to private
  4. Add team members as collaborators
  5. Clone the repository locally when the hackathon starts
This forking process is mandatory and will be used to verify that development started only after the hackathon's official start time. The original commit history from this template repository will be preserved.

How to Fork This Project - Step by Step Guide

  1. Login to GitHub
  2. Fork the repository
  3. Rename your forked repository to cubehack_yourprojectname
  4. Check repository visibility is private
  5. Add team members as collaborators
  6. Clone the repository locally
  7. Verify the setup

🔌 Technology Requirements

  • Cross-Platform: Flutter, React Native (Expo)
  • iOS (Native): Swift
  • Android (Native): Kotlin
  • A working version for both iOS and Android must be delivered.

☁️ Recommended Tools & Services

  • Database / BaaS: Firebase, Supabase, Appwrite
  • Push Notification: OneSignal, Firebase FCM
  • App Website (Optional): Firebase Hosting (e.g., *.web.app)

📋 Rules

  1. Teams may consist of up to 4 members.
  2. All development must start after the hackathon begins.
  3. The project must be a mobile application (Android, iOS, or cross-platform).
  4. The development process must be fully managed via GitHub.
  5. The submitted application must be functional and testable.
  6. All technologies used must be documented in the README.
  7. All designs must be made via Figma and the link must be provided.
  8. Code must be hosted on GitHub, properly structured, and well-commented.
  9. Repository name must follow this format: cubehack_projectname
  10. The repository must be private.
  11. Commit messages should be clear and meaningful.
  12. Development must reflect teamwork and collaboration.
  13. README must include participation terms, installation guide, and usage notes.
